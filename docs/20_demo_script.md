# Demo Script

## Demo Goal

Show that Casting AI is a complete industrial decision-support platform, not just a notebook or model.

## 5-Minute Demo Flow

### 1. Open the Dashboard

Say:

> This is the Casting AI quality monitor. It helps foundry engineers upload melting data, predict defect risk, detect anomalies, compare batches, and export QA reports.

Point out:

| Screen Area | What to Say |
|---|---|
| Upload section | The user can upload Excel or CSV. |
| Template download | The system provides a standard template. |
| Report Results | The dashboard produces production-style results. |

### 2. Upload or Load Data

Say:

> The platform validates the uploaded file before running ML. It checks headers, template compatibility, missing fields, extra fields, and feature schema alignment.

If using default data:

> For this demo, I am using the default processed dataset generated by the pipeline.

### 3. Explain Fleet Overview

Say:

> These cards summarize the whole fleet of castings. Fleet Quality and Process Stability show overall health. Average Defect Probability comes from the classifier. Critical Batches and Recommendations tell engineers where to focus first.

Point to:

| Card | Explanation |
|---|---|
| Fleet Quality | Overall quality signal. |
| Process Stability | Process consistency. |
| Dominant Severity | Most common anomaly level. |
| Avg Defect Prob | Average model risk. |
| Critical Batches | Immediate review count. |
| Recommendations | PROCEED, MONITOR, HOLD, STOP split. |

### 4. Explain Charts

Say:

> The donut and bar charts show severity and risk distribution. The gauges give an instant visual summary. The defect-driver charts show which process or chemistry parameters are most related to quality risk.

### 5. Explain ML Simply

Say:

> The model learned from historical batches labeled healthy or defective. For each new batch, it outputs a defect probability. Separately, anomaly detection checks whether the process pattern is unusual. The final recommendation combines both ML and foundry rules.

Use this simple analogy:

> The classifier asks, "Does this look like previous defective batches?" The anomaly detector asks, "Does this look unusual compared with normal production?" The rule engine asks, "Does foundry science say this condition is dangerous?"

### 6. Select One Batch

Say:

> Now I select one casting. The radar chart compares this casting against healthy fleet averages. This helps engineers see whether the selected batch differs in chemistry, temperature, shrinkage risk, gas risk, or Mg recovery.

### 7. Open Single Batch Analysis

Say:

> This is the row-level QA decision. It shows final risk score, recommendation, risk level, confidence, defect probability, anomaly score, key process parameters, and an engineering report.

Explain:

| Field | Demo Explanation |
|---|---|
| Final Risk Score | Strongest quantitative risk signal on 0-100 scale. |
| Recommendation | Operational action: proceed, monitor, hold, or stop. |
| Confidence | How strongly multiple signals support the decision. |
| QA Summary | Human-readable engineering explanation. |

### 8. Open ML Performance

Say:

> This page validates the model. It shows accuracy, precision, recall, F1, ROC-AUC, confusion matrix, ROC curve, precision-recall curve, feature importance, and confidence distribution.

Explain key metrics:

| Metric | Simple Explanation |
|---|---|
| Precision | When the model raises an alarm, how often it is correct. |
| Recall | How many real defects it catches. |
| F1 | Balance between precision and recall. |
| ROC-AUC | How well it separates healthy and defective castings. |
| False Negative | Defective casting missed; most dangerous case. |

### 9. Export Report

Say:

> Finally, the platform exports CSV, Excel, and PDF reports. This makes it useful for senior review, QA handover, and future traceability.
