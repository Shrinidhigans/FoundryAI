# Dashboard Guide

## Main Dashboard Pages

The dashboard is implemented with Streamlit. The most important user-facing pages are:

| Page | File | Purpose |
|---|---|---|
| Main Analytics | `dashboard/pages/main_analytics.py` | Unified upload, analysis, comparison, risk, and export workspace. |
| ML Performance | `dashboard/pages/ml_performance.py` | Model validation, metrics, explainability, and ML glossary. |
| Upload & Predict | `dashboard/pages/upload.py` | Dedicated batch upload and prediction flow. |

## Main Analytics

### Data Upload

This section allows users to upload a casting/melting Excel or CSV file. It also provides sample template downloads.

What to check:

| UI Item | Meaning |
|---|---|
| File uploader | Input for new melting log. |
| Sample Excel/CSV | Correct template format. |
| Template validation debug | Shows normalized, matched, missing, and extra columns. |
| Upload success message | Confirms number of processed castings. |

### Fleet Overview

This section summarizes the whole uploaded dataset.

| Card | Meaning | How to Interpret |
|---|---|---|
| Fleet Quality | Overall estimated quality score. | Higher is better. Low means many risky/defective/anomalous batches. |
| Process Stability | Stability across key process signals. | Lower score suggests process variation. |
| Dominant Severity | Most common anomaly severity. | CRITICAL/HIGH RISK means many unusual batches. |
| Avg Defect Prob | Average ML defect probability. | High value means the file contains many risky batches. |
| Critical Batches | Count of CRITICAL risk batches. | These require immediate engineering review. |
| Recommendations | Summary count of PROCEED/MONITOR/HOLD/STOP. | Useful for triage workload. |

### Fleet Charts

| Chart | Meaning |
|---|---|
| Anomaly severity donut | Distribution of NORMAL, LOW RISK, MEDIUM RISK, HIGH RISK, CRITICAL anomaly severity. |
| Risk level distribution | Count of LOW, MEDIUM, HIGH, CRITICAL final risk decisions. |
| Fleet quality gauge | Visual score of overall file health. |
| Process stability gauge | Visual score of process consistency. |

### Defect Drivers

This section explains what process or chemistry variables are most associated with defects.

| Chart | Meaning |
|---|---|
| Top defect-driving parameters | Largest differences or correlations between healthy and defective behavior. |
| Defect-driving features | Model feature importance or correlation fallback. |
| Parameter correlation heatmap | Relationships between chemistry and process variables. |

How to use it:

1. Look for features with high importance.
2. Compare them with known foundry control plans.
3. Check if drivers are chemistry, thermal, additive, or engineered risk features.

## ML Performance Page

### Model Performance Overview

| Metric | Meaning |
|---|---|
| Accuracy | Overall correctness. |
| Precision | How often defect alarms are correct. |
| Recall | How many real defects are caught. |
| F1 Score | Balance between precision and recall. |
| ROC-AUC | Separation quality between healthy and defective. |
| Total Samples | Hold-out evaluation samples. |
| Defective Samples | Known defective samples in test data. |
| Healthy Samples | Known healthy samples in test data. |

### Confusion Matrix

| Cell | Meaning |
|---|---|
| True Negative | Healthy casting predicted healthy. |
| False Positive | Healthy casting flagged as defective. |
| False Negative | Defective casting missed by model. |
| True Positive | Defective casting caught by model. |

False negatives are especially important because they represent escaped defects.

### ROC Curve

Shows true positive rate vs false positive rate across thresholds. A stronger model bends toward the top-left corner and has higher AUC.

### Precision-Recall Chart

Shows the tradeoff between catching defects and avoiding false alarms. Foundry QA often values recall strongly because missed defects are expensive.

### Feature Importance

Shows which model inputs contributed most strongly to predictions. Features are grouped into chemistry, thermal, additive, engineered, and other process categories.

### Confidence Distribution

Shows how many predictions are low, medium, or high confidence. Low-confidence cases are near the model decision boundary and should receive more engineering review.
